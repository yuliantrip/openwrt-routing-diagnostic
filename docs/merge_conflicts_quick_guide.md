# Быстрый гайд: как убрать конфликты PR и оставить актуальную версию из ветки Codex

## Что означает ваш пример

В конфликтном блоке:

```text
<<<<<<< codex/...
... (вариант из вашей ветки Codex)
=======
... (вариант из main)
>>>>>>> main
```

Если вы хотите оставить текущую доработанную версию скрипта (из ветки Codex), нужно оставить **верхний** блок (`<<<<<<< codex/...`) и удалить нижний (`main`).

## В GitHub UI (Resolve conflicts)

Поскольку UI-лейблы могут путать (`current`/`incoming`), ориентируйтесь по подписи в маркерах:
- сохранить то, что под `<<<<<<< codex/...`;
- удалить часть после `=======` до `>>>>>>> main`.

После этого:
1. Удалить сами маркеры `<<<<<<<`, `=======`, `>>>>>>>`.
2. Нажать **Mark as resolved**.
3. **Commit merge**.

---

## Самый безопасный способ (CLI)

Если умеете запускать git локально, так проще и быстрее:

```bash
# 1) перейти в ветку PR
git checkout <ваша_ветка_PR>

# 2) подтянуть main и смержить в ветку PR
git fetch origin
git merge origin/main

# 3) для конфликтных файлов оставить версию из вашей ветки (ours)
git checkout --ours owrt-diag.sh README.md docs/*.md

# 4) отметить resolved и закоммитить
git add owrt-diag.sh README.md docs/*.md
git commit -m "resolve merge conflicts: keep PR branch implementation"

# 5) запушить
git push
```

После пуша PR обычно автоматически покажет "no conflicts".

---

## Можно ли “просто применить последнюю версию целиком”?

Да. Это лучший путь для `owrt-diag.sh`, если вы хотите именно текущую реализацию из ветки PR:
- в конфликт-редакторе замените **весь файл** содержимым из ветки PR (без конфликт-маркеров);
- либо в CLI используйте `git checkout --ours owrt-diag.sh`.

Важно: после этого убедитесь, что нет строк с `<<<<<<<`, `=======`, `>>>>>>>`.
