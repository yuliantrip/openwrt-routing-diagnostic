# Как передать файлы скриптов в этот чат, чтобы я их точно увидел

Надёжные варианты (в порядке приоритета):

## 1) Вставить файл прямо в чат (лучший способ)

В одном сообщении:

```text
<filename: ssclash/clash-rules_3.7.0-r1_owrt25.12_cudy_slav.sh>
<BEGIN>
...полное содержимое файла...
<END>
```

Если файл большой — разбить на части:
- `PART 1/3`, `PART 2/3`, `PART 3/3`.

## 2) Добавить файлы в этот репозиторий (в текущую ветку)

Положить в путь:
- `ssclash/clash-rules_3.6.0-r1_owrt24.10_nanopi.sh`
- `ssclash/clash-rules_3.7.0-r1_owrt24.10_cudy_spask.sh`
- `ssclash/clash-rules_3.7.0-r1_owrt25.12_cudy_slav.sh`

После этого написать: «файлы добавлены, смотри `ssclash/`».

## 3) Дать raw-ссылки + короткий контроль доступа

Если через GitHub:
- ссылка должна быть `raw.githubusercontent.com/...`;
- репозиторий/файлы публичные;
- без auth/redirect ограничений.

Пример:
`https://raw.githubusercontent.com/<user>/<repo>/<branch>/ssclash/<file>.sh`

## 4) Архив (tar.gz/zip) в репозитории

Положить архив в `artifacts/` и написать путь.
Я распакую и сравню версии локально.

## Что удобно прислать дополнительно

Чтобы быстрее расширить диагностику:
- вывод `sha256sum` каждого файла;
- какой файл «рабочий» и какой «проблемный»;
- на каком роутере/версии OpenWRT использовался каждый скрипт.
